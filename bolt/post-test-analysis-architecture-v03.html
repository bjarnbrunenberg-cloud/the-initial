The flow · 9 nodes, top to bottom

1

Airtable Trigger

polling · 5 min

Filter: Status = Done AND Analysis status IS EMPTY. Emits: ID, Short name, Hypothesis, Primary Goal, Start / End date, Baseline Monthly Revenue, Omniconvert Test ID. Sets Analysis status = Processing.

2

HTTP Request — Omniconvert

data pull

GET /tests/{{Omniconvert Test ID}}/results. On failure the workflow-level error handler fires (not a counted node).

3

Code — Analyse

the brain JavaScript

One node does all the maths. Normalise variant names → z-test per metric → SRM check → verdict matrix → annualised impact.

Emits statsBundle: { verdict, primaryLift, sig, revenueLift, annualImpact, recommendedAction, metricsTable }
4

HTTP Request — Anthropic

narrative temp 0.3

Send statsBundle + Airtable context. System prompt enforces British English and "use only numbers from the bundle." Returns JSON: { conclusion, keyTakeaways[], recommendations[] }.

5

Google Slides — Create from template

deck

Copy template → rename BOLT-{{ID}}-PostTest-DRAFT. batchUpdate.replaceAllText on EXP_NAME, HYPOTHESIS, VERDICT, PRIMARY_LIFT, SIG, REVENUE_LIFT, ANNUAL_IMPACT, DURATION, CONCLUSION, KEY_TAKEAWAYS, RECOMMENDATIONS.

6

Slack — Send approval card

human gate

Block Kit card to #bolt-experiments. Header + verdict + primary / revenue lift + deck link + Approve / Reject buttons. action_id = approve | reject.

7

Wait — Webhook

pause

Path /webhook/post-test-approval. Resumes when Slack fires the interactive callback.

8

IF — action_id == "approve"

branch
TRUE → node 9

Continue to full Airtable write-back.

FALSE → reject path

Airtable Analysis status = Rejected. Slack thread reply "Rejected." End.

9

Airtable Update (on approve)

write-back

Result · Action Taken · Observed Uplift % · Statistical Significance · Learnings & Results · Next Steps · Analysis status = Approved · Analysis deck URL. Slack thread reply "Deck approved, Airtable updated."

Non-negotiables

  • • Nothing writes to final Airtable fields before human Approve.
  • • British English in all narrative.
  • • Percent fields pass decimals (0.0526, not 5.26).
  • • Never write formula fields: Exp name, Prio PXL, Running Time, Annualised Impact, Quality Score, ID.
  • • Result values exactly: Win | Learn (loss) | Inconclusive | Error.

Acceptance test — BOLT-12

Control: 128,933 users / 2,149 conv / £303,836 · Variant: 128,024 / 2,246 / £297,771 · 40 days.

Expected Conditional win — CVR ↑ sig, revenue ↓. If the workflow returns Clean Win, the verdict matrix in node 3 is wrong.
